# API Reference

nvoke.run exposes two surfaces:

- **Management API** — CRUD for functions, invocations, and API keys. Authenticated with a Clerk session token.
- **HTTP invoke** — the public `/f/:slug` endpoints your functions live on. Authenticated per-function (public or API key).

Start with the [overview](/api-reference/overview) for the base URL, authentication, and error shape common to every endpoint.
